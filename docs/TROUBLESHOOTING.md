# Lemon串口监控故障排查

[简体中文](TROUBLESHOOTING.md) | [English](TROUBLESHOOTING.en.md)

## 看不到任何数据

按这个顺序检查：

1. 点击“刷新端口”，确认目标 COM 端口存在。
2. 勾选目标端口。
3. 点击“开始”，确认按钮状态已经变化。
4. 在“开始”之后，让原业务软件真实发送或读取数据。
5. 确认原业务软件使用的 COM 号与勾选的一致。
6. 查看状态栏红色错误和驱动状态。
7. 列表只有 Ioctl 而终端为空时，说明只有配置操作，没有真实 Read/Write 正文。

软件不会为了测试而主动发数据。原业务软件没有通信时，监控界面没有数据是正常的。

## 端口列表为空或少了端口

- 如果设备管理器中也没有当前可用串口，列表为空是正常现象。后台服务应仍保持运行；状态中驱动暂不可用不等于服务启动失败。
- 确认设备管理器中能看到该端口。
- 拔插 USB 转串口后再点击“刷新端口”。
- 确认后台服务已启动。没有串口设备时，按需过滤驱动保持停止是正常状态；有设备仍无法就绪时再检查驱动服务和设备事件。
- 安装后尚未重启时先重启。
- 蓝牙、虚拟串口或厂商专用设备栈可能与标准 Ports 类不同，应保留硬件信息后单独验证。

## 服务未连接

用管理员 PowerShell检查：

```powershell
sc.exe query CommMonitorService
sc.exe qc CommMonitorService
sc.exe query CommMonitorFilter
```

服务未运行时：

```powershell
Start-Service CommMonitorService
```

如果马上再次停止，查看 Windows 事件查看器中的应用程序和系统日志。不要把服务改成普通用户运行，也不要从可写目录手工启动替代文件。

## 驱动未就绪

先确认电脑是否真的存在当前可用的串口设备。没有设备时，过滤驱动没有设备栈可绑定，界面显示驱动暂不可用是正常现象；接入设备后点击“刷新端口”再检查。

常见原因：

- 安装后没有重启。
- 安全启动仍开启。
- `TESTSIGNING` 尚未生效。
- 测试证书导入失败或驱动签名不匹配。
- 驱动包被安全软件或组织策略阻止。
- Ports 类过滤器没有正确加载。

管理员检查：

```powershell
bcdedit.exe /enum '{current}' | Select-String testsigning
sc.exe query CommMonitorFilter
```

不要启用 `nointegritychecks`，不要从未知网站下载驱动签名补丁。

## 安装程序提示安全启动开启

这是预期的安全门禁。安装程序不会替你关闭安全启动。

1. 先确认 BitLocker/设备加密恢复密钥可用。
2. 由电脑所有者决定是否在 UEFI 中关闭安全启动。
3. 不愿改变启动策略时取消安装，等待将来的正式签名版本。

每台电脑的 UEFI 界面不同，不能用通用脚本安全代替人工确认。

## 安装后提示需要重启

过滤驱动、Ports 类设备栈或 `TESTSIGNING` 发生变化时必须重启。保存工作后重启，再进行真实串口测试。不要在重启前反复卸载、重装或手工修改过滤器。

## 终端乱码

- 根据设备协议选择 ANSI、UTF-8、UTF-16LE 或 UTF-16BE。
- 改变编码只影响以后到达的数据；先清空界面或重新开始测试更容易比较。
- 二进制协议本来就不适合终端文本，改用列表 HEX 或 Dump。
- GBK/本地代码页通常选择 ANSI，但最终仍以设备协议文档为准。

## HEX 查找报错

正确：

```text
01 03 00 FF
03 ?? FF
```

错误：

```text
0x01 0x03
010300FF
GG
```

每个字节必须是两个十六进制字符，以空格分隔；`??` 只能代表一个字节。

## 复制后缺少端口或方向

HEX、文本、C 数组和 Python bytes 是纯负载格式，多行时会直接拼接。如果需要时间、端口、方向、进程和事件边界，选择 TSV、CSV 或 JSON。

`Ctrl+Shift+C` 固定复制纯空格 HEX，不包含元数据；需要当前选择格式时使用 `Ctrl+C`。

## 导出按钮不可用

导出只在停止状态启用。先点击“停止”，再检查导出文件名和格式是否填写。

导出的是完整持久化会话，不是选中的几行。只想复制选择内容时使用“复制数据”。

## AI 客户端无法连接

先运行：

```powershell
& '<安装目录>\ai\Lemon.SerialMonitor.AI.exe' status --json
```

检查：

- 后台服务是否运行。
- 是否使用安装目录中的原始 AI 客户端；复制到其他路径会被服务拒绝。
- 当前 Windows 用户是否为安装时授权的交互用户。
- 是否切换了用户或登录会话。
- MCP 配置中的 `command` 是否为绝对路径。
- JSON 中反斜杠是否写成 `\\`。

AI 接口不监听 TCP/HTTP，网络端口扫描看不到它是正常的。

## AI 读取提示完整性未知或有缺口

每一页都要检查：

- `integrity.completeForReturnedRange`
- `integrity.driverDropped`
- `integrity.serviceDropped`
- `integrity.truncationSeen`
- `integrity.gapDetected`
- `warnings`
- `nextCursor` 与 `resumeReceipt`

只有 `completeForReturnedRange` 为 `true` 时，才能把返回范围声明为完整。发生服务重启、游标失效、驱动丢弃、服务丢弃或截断时，应把结论标为不完整并保留相关证据。

## 原业务串口出现异常

优先恢复业务：

1. 在桌面程序中点击“停止”（如果还能操作）。
2. 关闭桌面程序。
3. 记录准确时间、COM 号、设备型号和业务软件错误。
4. 完整卸载 Lemon串口监控。
5. 按提示重启。
6. 验证原业务软件在没有本工具的情况下是否恢复。

不要反复重试、手工删除驱动服务或直接修改 Ports 类 `UpperFilters`。这些操作会破坏安装前基线，增加恢复难度。

## 卸载提示文件被占用

卸载程序会先关闭本软件桌面程序、AI 客户端和后台服务。若仍提示重启，通常是内核驱动、Ports 类设备栈、启动策略或 Windows 文件锁尚未释放，而不是要求用户手工结束本软件进程。保存工作并重启，登录后等待卸载完成。不要在重启前删除安装目录或取消计划任务。

如果重启后仍失败，保留提示文字、卸载日志、系统版本和状态脚本输出。受保护安装记录丢失时，卸载会停止以避免误删其他软件，这是安全行为。

## 收集诊断信息

提交问题时至少提供：

- Windows 版本、构建号、桌面体验或 Server Core
- 设备型号、USB 转串口芯片、COM 号
- 原业务软件名称和发生问题的操作
- 安装/重启/测试/卸载的准确时间线
- 桌面状态栏错误文字
- 管理员运行的状态脚本输出
- Windows 系统与应用程序事件
- AI 错误中的 `code` 和 `correlationId`
- 可公开的最小会话导出和 SHA-256（不要上传敏感业务数据）

需要严谨复现时，同时保存发送端日志、接收端日志、会话数据库/导出和文件哈希。

# Lemon串口监控操作指南

[简体中文](USER_GUIDE.md) | [English](USER_GUIDE.en.md)

## 先记住四件事

1. 软件只监控已经发生的通信，不会主动向设备发送数据。
2. 勾选端口以后还要点击“开始”。
3. 原业务软件必须在“开始”之后真实读写同一个 COM 端口。
4. 结束时先点击“停止”，再复制、导出或关闭窗口。

## 第一次使用

1. 启动原来使用串口的业务软件，按原来的方式连接设备。
2. 启动 Lemon串口监控（主程序文件名为 `Lemon.SerialMonitor.exe`）。
3. 点击“刷新端口”。左侧会列出当前检测到的 COM 端口。
4. 勾选一个或多个要监控的端口。
5. 在“会话”框填写文件名，例如 `board-test.db`。这里只填文件名，不填任意目录。
6. 点击“开始”。
7. 回到原业务软件进行发送、查询、升级或其他通信操作。
8. 在列表中确认出现事件。

同时监控多个端口时，多勾选几个端口再开始即可。软件不会因为监控而替原业务软件打开这些 COM 端口。

如果电脑没有接入任何串口设备，刷新后列表为空是正常现象。此时后台服务仍应保持运行，状态栏可显示服务已连接而驱动暂不可用；接入设备后再次刷新即可。

## 开始、暂停、继续、停止和清空

- **开始**：建立或继续写入指定会话，并开始接收所选端口的监控副本。
- **暂停**：业务串口仍照常通信，但暂停期间不记录监控副本。
- **继续**：恢复当前会话的监控记录。
- **停止**：结束当前捕获；已记录的数据和界面内容仍保留。
- **清空**：永久删除当前服务绑定会话中的记录并清空界面。该操作会要求再次确认。

“清空”不是普通的清屏。只想开始一份新记录时，先停止，改用新的会话文件名，再重新开始。

关闭桌面窗口不会自动替你按“停止”。养成“停止 → 导出/记录 → 关闭”的顺序。

## 会话自动保存

点击“开始”后，事件会持续写入本机受保护的会话数据库，不需要再按一次“保存”。默认会话名是 `capture.db`。

重复使用同一个会话文件名会继续写入该数据库。运行中修改会话名只影响下一次开始，不会把正在进行的捕获切换到新文件。

桌面界面主要显示当前连接后收到的实时事件；历史会话的自动化读取请使用 AI/命令行接口。需要交付给其他工具或人员时，使用“导出”。

## 三种视图

### 列表

列表适合逐条分析，包含：

- 序号与时间
- 发起操作的进程
- COM 端口
- Read、Write 或 Ioctl 方向/类型
- IOCTL 操作码与 NTSTATUS
- 完成长度与事件标志
- HEX 数据和文本预览

可以按住 `Ctrl` 或 `Shift` 选择多行。选中列表事件后，Dump 和终端会联动定位到同一个底层事件。

### Dump

Dump 每行显示 16 字节：左侧是偏移，中间是 HEX，右侧是可打印 ASCII。它适合核对协议帧、字段偏移、校验位和二进制内容。

### 终端

终端只显示 Read 和 Write 的正文，不显示纯配置 Ioctl。默认颜色：Read 为蓝色，Write 为红色。

可选编码：

- ANSI
- UTF-7
- UTF-8
- UTF-16LE
- UTF-16BE

还可以显示或隐藏时间、端口、方向，并切换自动换行和自动滚动。改变编码只影响之后到达的新数据，不会重新解释已经显示的内容。

## 查找

### HEX 查找

在查找类型中选择“HEX”，输入以空格分隔的字节：

```text
01 03 00 FF
```

`??` 表示任意一个字节：

```text
03 ?? FF
```

不使用 `0x` 前缀，也不要写成没有空格的 `010300FF`。点击“上一个”或“下一个”会从当前位置循环查找。

### 文本查找

选择“文本”后输入普通文字。搜索按 UTF-8 解释负载并忽略大小写。二进制协议或非 UTF-8 编码数据优先用 HEX 查找。

## 复制数据

这是软件中最常用的功能：

1. 在列表中选择一行或多行；也可以在 Dump/终端中选中对应事件。
2. 在“复制数据”右侧选择格式。
3. 点击“复制数据”，或按 `Ctrl+C`。

支持八种格式：

- **HEX（空格）**：`01 03 00 00 00 02`
- **HEX（紧凑）**：`010300000002`
- **文本**：按 UTF-8 复制负载
- **C 数组**：适合固件或 C/C++ 测试代码
- **Python bytes**：适合 Python 脚本
- **TSV**：适合直接粘贴到表格
- **CSV**：适合保存或导入数据工具
- **JSON**：保留事件字段，适合程序处理

快捷键：

- `Ctrl+C`：使用当前下拉框选择的格式。
- `Ctrl+Shift+C`：忽略元数据，只复制所选事件连续拼接后的空格 HEX。

选择多行时，HEX、文本、C 数组和 Python bytes 会按事件顺序拼接原始负载。需要保留端口、方向、时间和事件边界时，选择 TSV、CSV 或 JSON。

## 导出

导出只能在停止状态使用：

1. 点击“停止”。
2. 在导出文件框输入安全文件名，例如 `board-test.csv`。
3. 选择 CSV、TXT 或 RAW。
4. 点击“导出”。
5. 成功后，状态栏右侧显示输出位置。

导出的是当前服务绑定会话中的全部持久化记录，不是当前选中的几行，也不一定等于界面仍保留的可见行。

- **CSV**：包含结构化字段，适合 Excel、脚本和归档。
- **TXT**：便于人工查看的文本表示。
- **RAW**：按顺序连接原始负载，适合二进制再处理；不保留事件边界和元数据。

同名导出会替换旧文件。需要保留旧结果时先换文件名。

AI 接口另支持 JSON 和 JSONL 导出。

## 状态栏怎么读

- **服务**：是否连接到后台捕获服务。
- **驱动**：驱动或开发数据源状态。
- **事件**：本次界面累计接收的事件数。
- **丢失**：界面队列溢出或收到的丢失通知数。
- 右侧绿色文字：导出等操作的结果路径。
- 右侧红色文字：当前错误。

没有串口设备时，过滤驱动没有设备栈可绑定，“服务已连接、驱动暂不可用、端口为空”是正常组合，不表示服务崩溃，也不需要反复重装。

“丢失 = 0”不能单独证明整条链路绝对无丢包。需要严谨结论时，还应检查事件的 `Truncated`/丢失标志、AI 完整性字段、原业务日志和独立收发计数。

## 容量边界

- 单个内核捕获事件最多保存 4096 字节；更长数据会带截断标志。
- 列表保留最近 100,000 行，旧行从界面移除但已持久化事件仍在会话中。
- 终端保留约 2 MiB 可见文本，超过后裁掉最旧片段。
- UI 待处理队列最多 10,000 条，界面处理不过来时会记录丢失。

高吞吐或长时间测试优先使用会话数据库与 AI 分页读取，不要只依赖当前屏幕内容。

## 方向约定

- `Write` / `TX`：电脑发往设备。
- `Read` / `RX`：设备返回电脑。
- `Ioctl`：波特率、超时、线路控制、流控、DTR/RTS、清缓冲等控制操作。

## 日常结束检查

- 已点击“停止”。
- 已导出或复制需要的数据。
- 需要无损分析时，已记录会话名、时间范围和原业务软件日志。
- 完整卸载前，已把要保留的文件复制到本软件目录之外。

## AI 和自动化

AI、脚本或开发工具可在不打开桌面界面的情况下列出端口、控制捕获、读取历史会话、等待新事件和导出。接口只通过本机命名管道工作，不发送串口数据。

请从 [AI 接入指南](AI_INTEGRATION.md) 开始，字段和完整性判断见 [AI 接口参考](AI_API_REFERENCE.md)。

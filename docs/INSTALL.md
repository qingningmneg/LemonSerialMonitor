# Lemon串口监控安装与卸载

本文说明正式安装包的安装、重启、验证、迁移和完整卸载。普通使用者只需要双击安装包，不需要打开控制台。

## 安装前先确认

安装器接受以下 x64 兼容目标：

- Windows 10、Windows 11
- Windows Server 2019、2022、2025 桌面体验
- Windows Server 2019、2022、2025 Server Core

安装需要本机管理员权限。驱动当前使用本地测试证书，不是微软正式发布签名，因此还需要满足以下条件：

- 安全启动必须关闭；安装程序检测到安全启动开启时会停止。
- 如启用了 BitLocker 或设备加密，先确认恢复密钥可用。
- 安装程序可能启用 `TESTSIGNING`，该变更需要重启才能生效。
- 公司管理电脑应先取得管理员或安全团队许可。

安装程序不会关闭安全启动，不会暂停或修改 BitLocker，也不会使用 `nointegritychecks` 绕过 Windows 安全机制。

0.1.0 的安装文件使用本地测试签名，没有微软信任链，也没有 RFC 3161 公共时间戳。测试证书必须等安装程序取得管理员权限后才能导入，所以首次运行仍可能看到 SmartScreen 或“未知发布者”；这不是正式代码签名。只应从本项目 Release 下载并核对 SHA-256。

## 下载后先核对

只从本项目 GitHub Releases 页面下载安装包。发布页同时提供 `SHA256SUMS.txt`，可选地用 PowerShell 核对：

```powershell
Get-FileHash '.\Lemon串口监控-安装程序-x64.exe' -Algorithm SHA256
```

显示的 SHA-256 应与同一版本发布页一致。文件来源或哈希不一致时不要运行。

## 一键安装

1. 双击 `Lemon串口监控-安装程序-x64.exe`。
2. Windows 显示 SmartScreen 或“用户账户控制”时，只在下载来源和 SHA-256 与同版 Release 一致后继续。
3. 完整阅读“本地测试证书使用说明”，勾选接受后继续。
4. 选择桌面程序安装位置。默认是：

   ```text
   C:\Program Files\Lemon串口监控
   ```

5. 选择是否创建桌面快捷方式。
6. 在“准备安装”页核对安装模式、目标位置和测试签名说明。
7. 点击“安装”。安装程序会自动校验载荷、证书、系统、目录和现有安装状态。
8. 安装完成后按提示重启。需要重启时不要在重启前判断驱动是否可用。

安装过程中如出现 Windows 证书确认窗口，应把显示的证书指纹与同一次 Release 的 `BUILD-INFO.json` 核对；不一致时选择“否”并取消安装。

安装过程会自动完成以下事项：

- 校验安装包内所有文件的 SHA-256 清单。
- 校验驱动、目录文件和测试证书之间的签名关系。
- 把随包公钥证书精确导入本机 Root 与 TrustedPublisher 证书库。
- 在需要时启用 `TESTSIGNING`。
- 安装串口过滤驱动和 LocalSystem 后台服务。
- 把过滤器追加到 Ports 类过滤器列表，保留其他软件已有条目和顺序。
- 安装桌面程序、AI/命令行接口、文档、开始菜单入口和卸载程序。
- 写入受保护、带完整性校验的安装记录，供失败回滚和以后卸载使用。

安装脚本发现任何身份、路径、权限、哈希或签名不一致时会停止，并尽量回滚本次已经完成的步骤。

## 安装位置说明

安装向导中选择的是桌面程序位置。为防止普通用户替换高权限服务文件，以下目录由安装程序固定并使用受保护权限：

- 后台服务和驱动文件：`%ProgramFiles%` 下的内部核心目录
- 会话、导出和日志：`%ProgramData%` 下的内部数据目录
- 安装状态和卸载助手：`%ProgramData%\LemonSerialMonitor\Installer`
- 当前授权用户的 AI 租约：该用户 `%LocalAppData%\LemonSerialMonitor\AI`

这些内部目录不需要手工修改。桌面主程序文件名为 `Lemon.SerialMonitor.exe`。桌面版安装结束后，可从开始菜单或桌面快捷方式启动，也可以从开始菜单直接打开完整 PDF 操作手册。

## Windows Server

桌面体验版安装完整桌面程序、服务、驱动、AI 接口和文档。

Server Core 自动省略 WPF 桌面程序和快捷方式，只安装：

- 串口过滤驱动
- 后台捕获服务
- AI/MCP/命令行接口
- 状态、安装与卸载支持文件
- 文档

Server Core 使用 AI 命令行或 MCP 接口操作，见 [AI 接入指南](AI_INTEGRATION.md)。安装程序只接受 2019、2022、2025 的已知正式构建；未知 Server 构建会停止，避免把未验证版本当成兼容版本。

Windows Server 2022/2025 已在 GitHub 托管桌面 runner 完成平台识别、托管测试和安装契约检查，但没有装载内核驱动。Server Core 只有组件布局契约测试；Server 2019 自托管任务未执行。0.1.0 发布前没有任何 Server 实机或虚拟机的驱动安装、重启、捕获、AI、卸载端到端验收，因此这些版本只是兼容目标，不是已完成硬件认证。重要环境应先在同版本测试机完成全流程验证。

## 从旧的手工安装迁移

如果电脑上存在早期由本项目脚本安装的版本，安装程序会显示“迁移”模式。迁移只在旧安装记录、备份、服务路径、驱动包、过滤器和证书全部能相互验证时执行。

迁移过程会先建立受保护备份，再停止旧服务、移动旧文件并安装新版本。中途失败时会按事务记录恢复旧服务和文件。检测到身份不明确、多个候选驱动包或被修改的旧文件时，迁移会拒绝继续；不要手工删除记录后强行覆盖。

## 重启后验证

桌面系统：

1. 打开 Lemon串口监控。
2. 点击“刷新端口”。
3. 状态栏应显示服务已连接，驱动状态不应显示开发假数据源。
4. 勾选一个端口，点击“开始”，让原业务软件完成一次实际收发。
5. 列表中应出现 Read、Write 或 Ioctl 事件。

如果电脑当前没有串口设备，后台服务仍应保持运行。点击“刷新端口”后，显示“服务已连接”、端口列表为空、驱动暂不可用属于正常状态；接入真实串口设备后再次刷新，不需要为了这一状态反复重装。

管理员也可以运行随安装提供的只读状态脚本：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass `
  -File "$env:ProgramData\LemonSerialMonitor\Installer\scripts\Get-CommMonitorStatus.ps1"
```

状态应重点检查：平台、Secure Boot、TESTSIGNING、测试证书、Ports 类过滤器、驱动服务、后台服务、控制设备和是否需要重启。

## 正常使用中的更新

当前 0.1.0 安装器不在已有新式安装上执行原地覆盖。发布新版本时：

1. 停止监控。
2. 导出或备份需要保留的数据。
3. 使用旧版本卸载程序完整卸载。
4. 按提示重启并等待清理完成。
5. 安装新版本。

不要直接覆盖正在运行的服务、驱动或安装状态文件。

## 完整卸载

推荐入口：

1. 打开 Windows“设置”。
2. 进入“应用”→“已安装的应用”。
3. 找到“Lemon串口监控”。
4. 点击“卸载”。
5. 阅读数据删除警告并确认。
6. 允许管理员权限。
7. 如提示重启，保存工作后重启；重启后卸载程序会自动继续并核验结果。

管理员排查 Windows 设置入口异常时，也可以在提升权限后运行受保护安装器目录中的卸载程序：

```text
%ProgramData%\LemonSerialMonitor\Installer\unins000.exe
```

普通用户无法直接浏览该受保护目录是正常现象；日常卸载请始终使用上面的 Windows 设置入口。

完整卸载会永久删除：

- 本软件桌面程序、AI 客户端、文档和快捷方式
- 后台服务与串口过滤驱动
- 本软件加入的 Ports 类过滤器条目
- 本次安装实际拥有的测试证书
- 会话、导出、设置、日志、缓存和 AI 状态
- 安装记录、迁移备份和卸载续办任务

卸载只依据受保护安装记录和精确对象身份清理，不按模糊名称删除其他软件。服务路径、驱动 INF、证书指纹或文件身份不一致时会停止对应删除，避免误删外部对象。

如果安装程序是本次启用 `TESTSIGNING` 的，卸载会把它恢复为关闭并要求重启；如果安装前已经开启，卸载不会替用户改变原有策略。

卸载开始后会主动关闭本软件桌面程序、AI 客户端和后台服务。正常用户态文件不应仅因本软件自身仍在运行而要求重启；Ports 类过滤器、内核驱动、启动策略或 Windows 文件锁仍可能必须在重启后完成清理。

## 卸载前保留数据

最简单的方法是在桌面程序中停止监控后导出 CSV、TXT 或 RAW，并把导出文件复制到本软件数据目录之外。

需要保留完整会话数据库时，应先停止监控，再由管理员备份整个 Sessions 目录，包括同名的 `-wal` 和 `-shm` 文件。不要只复制主数据库后删除 WAL。

## 卸载异常

- 提示文件被占用：关闭桌面程序和使用相关文件的工具，然后按提示重启。
- 重启后仍有残留：不要手工删除服务或注册表。保留卸载日志和提示内容，按 [故障排查](TROUBLESHOOTING.md) 收集状态。
- 安装记录损坏或丢失：卸载会拒绝按名称猜测所有权。这种情况需要先恢复同版本受保护记录或进行人工核验，不能使用通配清理。
- 原串口在安装后异常：停止监控，完整卸载并重启；保留业务软件日志、状态输出和 Windows 系统事件用于定位。

## 安装日志

安装向导可在失败页查看错误。Inno Setup 安装日志、受保护事务结果和 Windows 服务/驱动事件是定位安装问题的主要证据。提交问题时请附：

- Windows 版本与构建号
- 桌面体验或 Server Core
- 安装/卸载发生的准确时间
- 安装向导完整错误文字
- 是否已重启
- 安全启动、BitLocker 与 `TESTSIGNING` 状态

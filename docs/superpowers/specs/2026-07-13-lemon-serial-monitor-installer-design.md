# Lemon串口监控图形安装器设计说明

- 日期：2026-07-13
- 状态：用户已确认交互设计、完整卸载、安全路径模型和品牌兼容方案
- 目标平台：Windows 10/11 x64
- 使用场景：个人自用，测试签名驱动

## 1. 目标

为现有串口监控系统生成一个真正面向普通用户的单文件图形安装包。用户双击安装程序、通过 UAC、选择客户端安装位置并点击“安装”后即可完成部署，全程不需要打开 PowerShell、Windows Terminal 或其他控制台。

产品对外名称统一改为 **Lemon串口监控**。安装包同时交付普通用户权限的 MCP/JSON AI 接口。安装器还必须提供可靠的图形卸载流程；完整卸载默认删除本产品安装的所有程序文件、AI 组件、系统组件、配置、日志、会话数据库和导出数据，并验证没有非预期残留。

## 2. 已确认的关键决策

1. 安装器采用官方稳定版 Inno Setup 6 作为图形外壳，复用并加固现有事务型 PowerShell 安装、状态和卸载逻辑。
2. 最终交付为单个 `Lemon串口监控-安装程序-x64.exe`。
3. 客户端目录由用户选择；默认值为 `%ProgramFiles%\Lemon串口监控`。
4. 以 LocalSystem 运行的服务、驱动支持文件、受保护清单和卸载安全元数据固定保存在 `%ProgramFiles%\CommMonitor`，不能放入普通用户可替换的目录。
5. 产品可见名称全部改为“Lemon串口监控”；内部服务、驱动和协议标识保持兼容。
6. 安装、迁移、修复和卸载均不显示控制台窗口。
7. 完整卸载默认永久删除 `%ProgramData%\CommMonitor` 中的 Sessions、Exports、日志、配置、备份和恢复数据。
8. 清理只能依据受保护的所有权清单执行，不使用 `*CommMonitor*` 等通配方式，不得影响本机已有的 CEIWEI `CommMonitor 12`。
9. 不自动关闭 Secure Boot，不静默修改 TESTSIGNING、BitLocker 或固件安全设置。
10. 驱动或锁定文件需要重启时，界面必须显示“重启后完成”，不能提前显示“卸载完成”。
11. 安装包包含 `Lemon.SerialMonitor.AI.exe`、AI 接入说明和 API 参考，但不自动修改第三方 AI 产品配置。

## 3. 方案比较与选择

### 3.1 选定方案：Inno Setup 6 + 事务型 PowerShell

Inno Setup 负责：

- 中文向导、UAC、目录选择、进度、完成和错误界面；
- 单 EXE 压缩与临时解包；
- 开始菜单、桌面快捷方式和“应用和功能”注册；
- 图形卸载入口、自身清理和重启提示。

PowerShell 负责：

- 安装前安全检查；
- 文件清单和散列验证；
- 证书、Driver Store、内核驱动、Ports 类过滤器和 LocalSystem 服务事务；
- 手工安装版本迁移、失败回滚、安全卸载和残留检查。

该组合保留了现有已经通过测试的系统修改边界，同时提供成熟的图形安装体验。

### 3.2 未选方案

- 自定义 WPF 安装器：需要自行重写安全解压、卸载注册、回滚、升级和重启恢复，风险和工作量明显更高。
- WiX/MSI/Burn：适合企业部署，但驱动、证书和复杂重启事务需要大量自定义提权操作，不适合当前个人交付目标。

## 4. 品牌与兼容标识

### 4.1 用户可见名称

以下位置统一显示“Lemon串口监控”：

- 主窗口标题和应用程序产品元数据；
- 安装器标题、欢迎页、进度页和完成页；
- 安装包文件名；
- 桌面快捷方式与开始菜单目录；
- Windows“应用和功能”显示名称；
- 服务显示名称和新事件日志源“Lemon串口监控服务”；
- 安装/卸载日志标题；
- 完整操作手册、安装说明和故障排查文档。
- AI 可执行文件、MCP 工具说明和 JSON CLI 帮助。

客户端发布文件使用 `Lemon.SerialMonitor.exe` 作为可见主程序名；现有 C# 命名空间可以保留，避免无意义的大范围代码改名。

### 4.2 保持不变和受控迁移的内部标识

为兼容当前已经运行的系统组件和数据，以下标识保持不变：

- Windows 服务内部名：`CommMonitorService`；
- 内核服务和过滤器名：`CommMonitorFilter`；
- 固定特权核心目录：`%ProgramFiles%\CommMonitor`；
- 托管数据目录：`%ProgramData%\CommMonitor`；
- 驱动设备及现有协议标识。

旧命名管道 `CommMonitor.Service.v1` 不安全地混合了控制和数据命令，升级时停止监听并原子迁移为 `Lemon.SerialMonitor.Control.v2` 和 `Lemon.SerialMonitor.AI.v1`。这些内部标识不作为主要用户界面文本，但完整卸载必须全部清除。

## 5. 目录与所有权模型

### 5.1 客户端目录 AppRoot

- 默认：`%ProgramFiles%\Lemon串口监控`。
- 用户可选择其他本地固定磁盘目录，包括 D 盘上的新目录。
- 目标必须为本地绝对路径，不得是卷根、UNC、设备路径、网络盘、移动盘、重解析点、符号链接或联接点。
- 目标必须不存在或为空；不覆盖含有未知文件的目录。
- 客户端不以管理员或 LocalSystem 身份运行，因此可以位于普通用户可写的本地目录。
- 因 AppRoot 可能由普通用户修改，提权卸载不能对它执行基于字符串路径的递归删除；必须使用第 5.6 节的受保护卸载助手逐项验证和删除。

### 5.2 特权核心目录 CoreRoot

- 固定：`%ProgramFiles%\CommMonitor`。
- 包含后台服务、AI 专用管道服务、驱动支持文件、事务脚本、受保护安装清单和安全卸载元数据。
- 所有者和 ACL 仅允许 SYSTEM/Administrators 修改，普通用户只读执行。
- CoreRoot 及其现有父目录必须通过重解析点和 ACL 安全检查。

### 5.3 数据目录 DataRoot

- 固定：`%ProgramData%\CommMonitor`。
- 包含 Sessions、Exports、安装备份、事务备份、恢复记录和运行日志。
- 服务运行所需子目录采用受限 ACL。
- 完整卸载默认删除整个经受保护清单验证的 DataRoot。

### 5.4 安装器自身 InstallerRoot

Inno 自身卸载文件、已签名的 `Lemon.UninstallHelper.exe` 和持久化卸载入口存入独立且受保护的 `%ProgramData%\LemonSerialMonitor\Installer`。InstallerRoot 不放在 DataRoot 或 AppRoot 内，避免事务卸载器删除 DataRoot 时与正在运行的 `unins*.exe` 冲突。无需重启时，事务卸载器先清理系统组件、AppRoot、CoreRoot 和 DataRoot，再由 Inno 在退出时删除精确 AppId、InstallerRoot 和自身；需要重启时改用第 8.3 节的延迟完成状态，不能提前删除助手或卸载入口。

### 5.5 受保护所有权清单

清单至少记录：

- 固定且不可变的产品 AppId；
- InstallId 和版本；
- CoreRoot、AppRoot、DataRoot、InstallerRoot 的规范化路径；
- 本次安装创建的每一个产品文件、AI 组件、快捷方式和注册表入口，并区分“带发布 SHA-256 的不可变安装对象”和“固定相对路径的产品动态状态”；
- 用户服务的精确 ImagePath；
- Driver Store 发布名、原始 INF 路径和 INF SHA-256；
- 证书指纹以及 Root/TrustedPublisher 是否由本次安装新增；
- 事件日志源安装前是否存在、是否由本次安装创建；
- AI 管道授权的原始交互用户 SID；
- 该 SID 对应的规范化用户 Profile/LocalAppData 根和本产品 AI 状态相对路径；
- AI sessionId/cursor 密钥元数据的精确位置和 keyId；
- 安装前 Ports `UpperFilters` 的精确状态，仅用于本次安装失败的事务回滚；
- 需要重启完成的操作。

清单保存在受保护位置并带完整性校验。卸载不能信任用户可写 AppRoot 内的路径声明。

### 5.6 高权限安全删除模型

所有涉及普通用户可写位置的清理由受保护、已签名的 `Lemon.UninstallHelper.exe` 完成。助手只读取 CoreRoot/InstallerRoot 中通过完整性验证的清单，绝不读取 AppRoot 内提供的删除路径，并遵守：

- 以 `FILE_FLAG_OPEN_REPARSE_POINT` 和目录句柄逐级打开 AppRoot、用户 AI 状态根及每个清单相对路径；
- 对每一级检查 reparse 属性，使用 `GetFinalPathNameByHandle` 验证最终对象仍在已批准根内；发现 symlink、junction、mount point、未知 reparse tag 或路径身份变化立即停止该项；
- 对不可变安装文件校验清单相对路径、普通文件类型、产品标记和 SHA-256；内容或身份已被用户替换时不以管理员权限猜测删除，只报告残留；
- 对 `leases.json` 等运行期动态状态，安装时无法预知内容散列；只允许清单声明的固定产品相对路径和文件名，在句柄确认它是根内普通文件、不是 reparse point 且父目录身份未变后删除，不要求匹配安装时 SHA-256，也不能扩大为目录通配；
- 使用句柄删除语义删除已验证文件，不能先检查字符串路径后再调用 `Remove-Item -Recurse`、`Directory.Delete(..., true)` 或其他会跟随目录链接的 API；
- 目录只在从叶到根重新以句柄确认无 reparse point 且为空时删除；未知文件和目录保留并在结果页列出；
- 每次验证到删除之间保持对象句柄，避免普通用户在检查后替换路径产生 TOCTOU；
- DataRoot、CoreRoot 和 InstallerRoot 虽受 ACL 保护，仍执行根句柄、最终路径和清单身份检查。

授权用户的 AI 租约文件从 Windows ProfileList 中按记录 SID 推导，不信任进程环境变量。其 `%LocalAppData%\LemonSerialMonitor\AI` 目录使用同一不跟随重解析点的逐项删除模型。

## 6. 单文件安装流程

### 6.1 启动和预检

1. 用户双击 `Lemon串口监控-安装程序-x64.exe`。
2. 安装器自动请求管理员权限；不弹出控制台。
3. 在进行任何系统修改之前检查：
   - Windows 10/11 x64 和 64 位 PowerShell；
   - 安装包内部清单、SHA-256 和驱动签名器指纹；
   - CoreRoot、DataRoot 和候选 AppRoot 的路径安全性；
   - AI 管道将授权的原始交互 Windows 账号；UAC 使用其他管理员账号时必须显示并确认实际授权对象；
   - 可用磁盘空间；
   - Secure Boot 与 TESTSIGNING 状态；
   - 同名服务、驱动、手工安装或图形安装版本；
   - 是否存在不属于本产品的冲突组件。
4. Secure Boot 开启或 TESTSIGNING 未启用时，在修改系统前停止，并显示中文原因和操作手册入口；不自动更改启动安全策略。

### 6.2 向导页面

向导保持简短：

1. 欢迎与环境检查；
2. 客户端安装位置；
3. 可选桌面快捷方式；
4. 安装摘要；
5. 安装进度；
6. 完成、立即重启或稍后重启。

所有 PowerShell 调用均使用 `-File` 和独立参数，通过隐藏的 64 位 PowerShell 启动。安装器捕获标准输出、错误输出和退出码，并映射为中文状态。

### 6.3 文件和系统事务

1. Inno 只将完整 payload 解压到随机临时目录，不预先把文件复制到 AppRoot 或 CoreRoot。
2. 安装脚本验证 payload 的 SHA-256 清单。
3. 创建受 ACL 保护的事务记录。
4. 把客户端复制到 AppRoot，把服务、驱动支持文件和安全元数据复制到 CoreRoot。
   AppRoot 同时包含 `Lemon.SerialMonitor.AI.exe` 及其普通用户依赖，CoreRoot 包含 AI 专用管道服务代码。
5. 仅在备份原状态后导入测试证书、添加 Driver Store 包、创建内核服务、修改 Ports `UpperFilters`、创建 LocalSystem 服务。
6. PowerShell 事务成功后，由 Inno 创建精确快捷方式和固定 AppId 的卸载注册项；Inno 在后续失败时负责回滚这些外壳项目。
7. 任一步失败时按事务记录逆序回滚；如果系统事务成功后 Inno 外壳步骤失败，Inno 必须调用同一受保护回滚入口撤销系统修改。界面随后显示失败原因和日志路径。
8. 成功后提示必须重启才能确保过滤驱动对所有串口设备生效。

## 7. 当前手工安装版本迁移

本机当前已有受保护标记的手工安装，且 `CommMonitorService` 正在运行。图形安装器必须支持安全接管，而不是要求用户运行控制台卸载命令。

迁移流程：

1. 验证现有安装标记、事务备份、服务 ImagePath、驱动发布名和散列。
2. 只有所有身份完全匹配时才进入“迁移到 Lemon串口监控安装器管理”。
3. 保留当前驱动和 Sessions/Exports 数据，不进行破坏性卸载；若当前正在捕获，先要求停止并确认 SQLite 已静止。
4. 事务备份现有服务文件，短暂停止 `CommMonitorService`，原子升级服务以加入 AI 管道；新服务启动和状态检查失败时恢复旧服务文件并重新启动。
5. 把客户端和 AI EXE 复制到用户选择的 AppRoot，关闭旧客户端后移除已验证的旧客户端文件。
6. 把服务显示名称和事件日志源迁移到“Lemon串口监控服务”；只有确认旧事件源属于本安装时才移除旧源。
7. 扩展受保护清单，记录 AppRoot、AI 授权 SID、品牌信息、快捷方式和固定 AppId。
8. 注册图形卸载入口并验证新客户端和 AI CLI 都能连接升级后的服务。
9. 如果检测到未知或被篡改的同名服务/驱动，拒绝接管并显示精确原因，绝不猜测或强制删除。

## 8. 完整图形卸载流程

用户可从 Windows“应用和功能”或开始菜单启动卸载。卸载器请求 UAC，但不显示控制台。

### 8.1 数据删除确认

卸载页明确以红色文字说明：

> 完整卸载将永久删除全部监控会话、数据库、导出文件、配置和日志，且无法恢复。

用户必须确认后才能继续。按已确认需求，不提供默认保留数据的行为。用户此前另行复制到产品目录之外的文件不属于本产品所有，不予删除。

### 8.2 安全卸载顺序

1. 验证管理员权限、固定 AppId、受保护清单和事务备份。
2. 仅关闭清单记录的 `Lemon.SerialMonitor.exe` 和 `Lemon.SerialMonitor.AI.exe` 进程并停止 `CommMonitorService`。
3. 先事务记录卸载开始时 Ports 类 `UpperFilters` 的当前快照，再只移除大小写不敏感的精确 `CommMonitorFilter` 项；验证结果等于“卸载前快照删除该精确项”，其他软件后来增加的过滤器及其顺序必须保持不变。安装前快照只供安装失败回滚，不能在正常卸载时恢复。
4. 删除并验证用户服务。
5. 根据受保护的发布名、原始 INF 路径和散列调用 `pnputil /delete-driver ... /uninstall /force`，只接受成功或明确的 3010 重启状态。
6. 删除并验证仅由本次安装创建的内核服务。
7. 只删除本次安装新增的 LocalMachine Root 和 TrustedPublisher 证书条目；不删除安装前已有的证书，也不删除构建阶段创建的 `CurrentUser\My` 私钥证书。
8. 只删除受保护清单证明由本安装创建或迁移拥有的 Lemon/旧 CommMonitor 事件日志源注册键；不清空共享的 Windows Application 历史日志。
9. 删除清单记录的精确桌面和开始菜单快捷方式。
10. 删除 Sessions、Exports、临时文件、日志、备份和恢复记录，再删除整个经验证的 DataRoot。
11. 由受保护卸载助手按第 5.6 节逐句柄删除 AppRoot 和授权用户 AI 状态根中清单拥有的产品文件；若用户后来替换文件或放入非清单文件，只删除能够重新验证的产品对象并明确报告，不能误删未知对象。
12. 删除 CoreRoot 中经标记验证的文件和目录。
13. 无待重启操作时，由 Inno 删除固定 AppId 的卸载注册项、InstallerRoot 和自身；存在待重启操作时进入第 8.3 节状态并保留最小完成组件。

### 8.3 失败与重启状态

- `sc.exe`、`pnputil.exe` 或证书删除的非预期失败必须传播为卸载失败，不能继续显示成功。
- `MoveFileEx` 只可用于 ACL 受保护的 CoreRoot、DataRoot 或 InstallerRoot 内已通过第 5.6 节验证的对象。AppRoot 和授权用户 LocalAppData 等普通用户可写根禁止登记基于字符串路径的重启删除；锁定对象留到启动任务运行时重新逐级取句柄、验证并立即删除。
- 需要重启时显示“重启后完成卸载”，而不是“卸载完成”。
- 如需启动后复核，InstallerRoot 暂时只保留已签名卸载助手、Inno 最终清理器、不可变完成清单和最小诊断日志；固定 AppId 卸载项原子改为调用 Inno 最终清理器的 `/resume <InstallId>`，显示名称标记为“Lemon串口监控（等待重启完成卸载）”。不能在此阶段宣称卸载成功。
- 创建一次性、精确 InstallId 命名且操作定义不可被普通用户修改的 SYSTEM 清理任务，任务入口同样是 Inno 最终清理器 `/resume <InstallId>`。最终清理器作为父进程启动卸载助手复核服务、驱动包、过滤器、待删除文件和路径身份，并等待助手完全退出；助手成功后写入绑定 InstallId 的认证完成令牌。
- 最终清理器验证完成令牌后删除已退出的助手、任务、固定 AppId、清单和日志，再使用 Inno 自带的受控自删除阶段清除 InstallerRoot 和自身。助手不能直接删除正在运行的自身，最终清理器也不能被单独调用来删除任意路径；任一步失败都保留可重试入口和诊断状态。
- 启动后复核失败时保留受保护的最小助手、清单、日志和可重试卸载入口，界面显示具体残留；下次运行从已验证状态幂等重试，不能留下无入口的半卸载状态。
- Windows 共享 Application 历史事件、SetupAPI 日志、Prefetch 和 Amcache 属于系统审计历史，不是本软件安装文件，不进行破坏性清理。

## 9. 残留验证

完整卸载并重启后必须全部满足：

- `CommMonitorService` 不存在；
- `CommMonitorFilter` 不存在于系统驱动服务和服务注册表；

- 本次新增的 LocalMachine Root/TrustedPublisher 证书不存在；
- 本次创建的事件日志源键不存在；
- AppRoot、CoreRoot、DataRoot、InstallerRoot 和授权用户 LocalAppData AI 状态根中本产品拥有的文件不存在；
- 安装器创建的 AI 配置样例不存在；
- 产品快捷方式、固定 AppId 卸载项、一次性任务和 Run/RunOnce 项不存在；

- `\\.\Global\CommMonitorFilter` 控制设备无法打开；

- CEIWEI `CommMonitor 12` 的安装目录、快捷方式、卸载项和运行状态未改变。

## 11. 签名与安全边界

- 驱动继续采用个人测试证书和 Windows 测试签名模式。

- 安装器不绕过 Secure Boot，不自动修改 BCD/TESTSIGNING，不静默改变 BitLocker 状态。
- 所有 native 命令都使用参数数组或安全转义，不能通过拼接 `-Command` 执行用户路径。

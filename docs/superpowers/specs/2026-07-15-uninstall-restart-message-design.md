# 卸载重启提示准确性设计

日期：2026-07-15
状态：已确认

## 背景

实际卸载证据表明，应用进程、后台服务和内核服务均已退出或删除。Windows `setupapi.dev.log` 同时记录驱动文件未被任何进程占用、驱动包删除成功，但 Driver Store 仍将内核驱动清理标记为 `Delete pending`。现有提示把所有 `PendingReboot` 情况概括为“文件仍被 Windows 占用”，会让用户误以为应用没有关闭自己的进程。

## 方案

保留现有安全事务和重启续办机制，不尝试强制重启或禁用其他串口设备。仅将提示改为：

> Windows 正在完成驱动或设备栈的安全清理。请重新启动计算机，卸载会自动继续。

该文字覆盖驱动包、PnP 设备栈和受保护目录延迟清理等受支持的 `PendingReboot` 来源，不再断言存在用户态文件占用。

## 范围

- 修改 Inno Setup 卸载提示。
- 增加回归测试，要求新提示存在，并禁止旧的误导性表述重新出现。
- 在安装、卸载和故障排查说明中解释：重启可能用于完成内核驱动或设备栈切换，不代表应用进程仍在占用文件。
- 不改变卸载结果 JSON、退出码、续办任务、驱动删除命令或数据删除规则。

## 验证

1. 先添加测试并确认它因旧提示而失败。
2. 修改最少量的安装器文字，使测试通过。
3. 运行完整 Pester 套件、公开名称审计和安装器编译。
4. 用新安装包复跑安装、卸载、重启续办与零残留验收。

## 成功标准

- 用户不再看到“应用未关闭自己文件”的错误暗示。
- 必要重启仍按原有安全流程执行，续办任务和零残留结果不受影响。
- 自动化测试能阻止旧文案回归。
